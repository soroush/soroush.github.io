نصب MySQL Server با استفاده از Inno Setup

۲۲ اسفند ۱۳۹۲

خیلی زیاد پیش میاد که توی برنامه‌هامون از پایگاه داده استفاده کنیم. معمولاً برای کارهای کوچیک از سیستم‌های پایگاه داده‌ای قابل حمل مثل SQLite استفاده می‌کنیم SQLite در واقع یک کتابخانهٔ سی هست که بلده توی یک فایل داده‌ها رو به‌شکل ساختارمند بنویسه. بنابراین مشکلی نخواهیم داشت اگه بخواهیم برنامه‌ای رو که با استفاده از SQLite نوشتیم به کامپیوتر کاربرامون منتقل کنیم. اما مشکلی که در مورد SQLite وجود داره اینه که برای دامنهٔ وسیعی از کاربردها مناسب نیست. سرعتش خیلی کمه و محدودیت‌های جدی داره. در این موارد چاره‌ای نداریم از سیستم‌های مدیریت پایگاه داده‌ای پیشرفته‌تر (و پیچیده‌تر) مثل MySQL یا PostgreSQL استفاده کنیم. این سیستم‌ها خیلی قدرتمند و بی‌عیب و نقص هستند و اصولاً برای کار با حجم بسیار بالای داده‌ها ساخته شدند. اما مشکل اصلی همین‌جا شروع میشه! سیستم مدیریت پایگاه داده یک برنامهٔ کاملاً مستقل هست که معمولاً به‌شکل سرور باید روی کامپیوتر مقصد (کامپیوتر کاربر) نصب بشه و معمولاً پیکربندی نه‌چندان آسانی داره.

توی این پست در مورد نحوهٔ نصب و پیکربندی سرور MySQL به‌شکل مخفیانه (و بی‌سروصدا) روی کامپیوترهای کاربران می‌نویسم. با استفاده از این روش خیلی آسون می‌تونید MySQL رو موقع نصب برنامهٔ خودتون، روی کامپیوتر کاربر نصب کنید و تنظیمات اولیه‌ش رو انجام بدید بدون این که روح کاربرمون خبردار بشه :) (البته شوخی کردم، براساس احترامی که برای کاربران قائل هستیم، یک سری پیغام‌هایی وسط پروسهٔ نصب نمایش داده میشه)

خوب اول کار باید بگم که ما از سیستم نصاب Inno Setup استفاده می‌کنیم. اینوستاپ یک برنامهٔ متن‌باز بسیار عالی برای ساخت پکیج‌های نصب روی ویندوز هست. معمولاً برای تمام کارهای ویندوزی‌م از این برنامه استفاده می‌کنم. برنامه‌های نصب پروژهٔ EBO-Qt رو هم با همین برنامه درست کردم.
